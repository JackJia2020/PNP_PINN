# 1D Poisson-Nernst-Planck (PNP) Benchmark: PINN vs. Supervised NN

**Author:** Jack  

## Overview
This repository contains a comparative study evaluating the efficacy of Physics-Informed Neural Networks (PINNs) against standard, purely data-driven Neural Networks for solving the 1D Poisson-Nernst-Planck (PNP) equations. The system models the drift-diffusion dynamics of cations and anions under an electric field with blocking electrodes.

The project is divided into three main components:
1. **FDM Ground Truth:** A classical explicit Finite Difference Method (FDM) solver that generates high-fidelity baseline data.
2. **PINN Model:** A model trained purely by minimizing the PDE residuals (Nernst-Planck and Poisson) and enforcing physical boundary constraints, using the `physicsnemo.sym` framework.
3. **Supervised NN Model:** A baseline neural network trained explicitly on an 80% split of the FDM auxiliary data using Mean Squared Error (MSE).

## Installation & Requirements
To run the code locally, ensure you have Python installed. It is recommended to use a virtual environment or an Anaconda environment.

```bash
pip install torch numpy pandas matplotlib scipy sympy physicsnemo
```
## How to Run the Pipeline
The repository is designed to be fully reproducible. Run the following commands from the root directory:
1. Generate the Ground Truth Benchmark
2. Run the explicit Finite Difference Method (FDM) solver to generate the numerical baseline.

```bash
python src/FDM_solver.py
```
3. Train the PINN
Train the Physics-Informed Neural Network using the physicsnemo.sym framework.

```bash
python src/pnp.py
```
4. Train the Supervised Neural Network
Train the purely data-driven model.

```bash
python src/PNP_supervised.py
```

## Key Findings

1. **Superior Accuracy of Supervised Learning:** The standard Neural Network (NN) was much more accurate than the PINN. The highest error for the NN was only about 0.05, which is much lower than the PINN's error of 0.27.

2. **Architecture Capability:** Because the standard NN performed so well, it proves that our network architecture (8 layers, 256 neurons) is definitely capable of solving these equations. The core structure of the model is not the problem.

3. **PINN Optimization Challenges:** The PINN understands the general physics and the "shape" of the solution, but it struggles with exact numbers. This suggests that training a model using only physics laws is a much harder task for the computer than learning directly from data.

4. **Potential Field ($\phi$) Accuracy:** The standard NN perfectly reconstructed the electrical potential and boundary values. The PINN showed more errors in the middle of the domain, likely needing more training time or better settings to match the benchmark.
